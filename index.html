Understanding Git and GitHub

Git

Learn how to use Git bash (Terminal). Check most useful commands like:

  • $ pwd - Checks where terminal is open
  • $ cd ~/Path/To/Directory - Changes directory
  • $ ls - Shows the files and folders list in the directory
  • $ mkdir dirname - Creates new directory
  • $ rm -r dirname - Deletes directory or file
  • $ git init - Initializes directory
  • $ git status - Checks directory files if changed
  • $ git diff - Shows difference between old and new versions.
  • $ git add filename.html - Adds file to stage
  • $ git add . - Adds all files to stage
  • $ git rm --cached filename - Unstages files
  • $ git commit -m "Message" - Commits changes with message
  • $ git log - Shows log

How Branching works. Create new branches, to work with different versions, merge or delete current branches.

  • $ git branch - Checks which branch is active, shows branch list
  • $ git branch branchname - Creates new branch
  • $ git checkout branchname - Switches the branch
  • $ git checkout -b branchname - Creates new branch and switches it
  • $ git merge brancname - Imports changes from version in the current branch
  • $ git branch -D branchname - Deletes the branc
  • $ git -m currentbranchname newbranchname - Renames the branches
  • $ git remote add origin /URL/ $git push -u origin master - Pushes local repo to remote
  • $ git clone --mirror githuburl.git - Clone remote version of github project locally*
*URL can be copied from GitHub website.

Github

GitHub is a publishing tool, version control system and collaborating platform for web developers and designers. Basic understanding of how Git works and its basic commands are critical in using GitHub correctly. Learning these basics first will speed up your ability to learn GitHub and use it correctly.

Start using GitHub by sign up. Create your first repository (directory). Fill the readme, gitignore and license files. Clone remote version wuth Git bash by url or GitHub client application. GitHub client - is a desktop application by GitHub. It helps developers do same things as in Git. You can clone your remote repository locally, push updated version on GitHub site, compare local and remote versions on GitHub, stage and commit new changes, or work only locally and don't publish files on GitHub. When you will install application, must login with your GitHub account credentials, to configure it. In the options (ctrl+, ) advanced tab you can choose external text editor, Shell options - Shell, Bash or PowerShell. See all keyboard shortcuts HERE. When you create new project, GitHub client asks you if want to create new repository. It means that your project folder will be added in Git envaironment. Application has branch control options, where can create, merge and organise different branches. GitHub client has developer tools pane, like as in browser, where you can check errors and warnings, test new changes etc...

All in one - Atom

Atom Text Editor

Atom is a free and open-source text and source code editor for macOS, Linux, and Microsoft Windows with support for plug-ins written in Node.js, and embedded Git Control, developed by GitHub. Inside Atom there are integrated Git and GitHub panes, where are possibility to do same functions as in Git bash and/or GitHub desktop application. Git pane allows you to stage new changes, commit with message. GitHub pane gives opportunity to check pull requests, checkout or create new branches.

Atom has push, pull and fetch function like GitHub desktop application. Fetch button compares remote and local versions of repository and gives you ability to pull remote version if it is newest, or if you make changes locally, then you can use push function to upload new files on GitHub remote directory, after changes will be commited by git.
